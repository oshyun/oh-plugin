# Doc Style

## 문서화 습관

- **문서는 줄을 짧게.** 한 불릿·문장에 몰아넣지 말고 짧게 끊어 별도 줄로 분리한다.
  - 성격이 다른 내용(동기 vs 동작)은 줄을 나눈다.
- **코드 변경 시 사용자 가이드 동기화 필수.** 기능·동작·UI·플로우가 바뀌면 사용자 가이드/문서도 **같은 작업에서** 갱신한다.
  - API 문서 갱신과 동급으로 취급. 문서 상단의 업데이트 날짜도 함께 갱신.
- **API 변경 시 API 문서 동기화.** 엔드포인트 추가/수정/삭제 시 API 문서를 함께 업데이트한다.
